## Journal Reflection

### 1) Client summary and software requirements
Artemis Financial is a financial consulting company that provides personalized financial plans. The client wanted to improve the security of their web application and protect sensitive customer data during communication. The main issue to address was identifying vulnerabilities and implementing secure coding practices to reduce risk and improve trust.

### 2) What I did well and why secure coding matters
I did well at analyzing the application for security weaknesses and recommending practical mitigations that align with industry best practices. Secure coding is important because vulnerabilities can lead to data breaches, financial loss, downtime, and loss of customer trust. Strong software security improves a company’s reliability, reduces risk, and supports compliance expectations.

### 3) Most challenging or helpful part of the vulnerability assessment
The most challenging part was prioritizing findings by severity and understanding the impact of each issue in a real-world context. The most helpful part was using structured tools and reporting to clearly identify what needed to be fixed and why.

### 4) Layers of security and future approach
I increased layers of security by applying defense-in-depth concepts such as stronger dependency management, secure communication protections, and improved validation and configuration practices. In the future, I would continue using tools like static analysis, dependency scanning, and manual review to assess vulnerabilities, then choose mitigations based on risk, impact, and feasibility.

### 5) Ensuring the application stayed functional and secure
To ensure the software remained functional and secure, I tested the application after changes and verified that security improvements did not break expected behavior. After refactoring, I re-ran security scans (including dependency checks) to confirm that new vulnerabilities were not introduced and that previously identified issues were addressed.

### 6) Resources, tools, and practices used
Helpful resources and tools included OWASP guidance, dependency scanning (OWASP Dependency-Check), Maven build verification, and secure coding best practices like input validation, principle of least privilege, and secure configuration. These tools and practices will help in future assignments and real-world development.

### 7) What I would show future employers
I would show future employers the completed secure software practices report and the evidence of scanning and verification steps because it demonstrates I can identify vulnerabilities, communicate security risks clearly, and apply mitigations while keeping software stable and maintainable.
